设计模式之工厂模式

—-逃梦

定义

  工厂模式。

  英文全称：Factory Pattern。

  简称：FP

  英文定义：Define an interface for creating an object , but let subclasses decide which class to instantiate , Factory Method lets a class defer instantiation to subclasses.

  中文解释：定义一个用于创建对象的接口，让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类。

实现思想

  通用工厂模式包含：抽象产品类(或接口)、实现产品类、抽象工厂类，实现工厂类。

优点

  • 良好的封装性，代码结构清晰，降低模块间的耦合；
  • 很好的扩展性，在增加产品类的情况下，只需要适当修改具体的工厂类或扩展一个工厂类，即可完成变化；
  • 对高层模块屏蔽产品类，高层只需要关心产品的接口，只要接口不变，高层不需要发生变化；
  • 工厂模式是典型的解耦框架，高层模块只需要知道产品的抽象类，其它实现类不用关心，不需要的不产生通信，符合迪米特法则；只依赖产品类的抽象，符合依赖倒置原则；产品子类可以替换产品父类，符合里氏替换原则。

缺点

  • 增加工厂类进行管理，增加了代码的复杂度
  • 扩展种类很多，各有优缺点

使用场景

  • 所有需要生成对象的地方都可以考虑使用；
  • 需要灵活、便于扩展的框架时，可以考虑使用；
  • 异构项目中，可以考虑使用工厂模式进行管理，减少与外围系统的耦合；
  • 测试驱动开发的框架下，可以考虑工厂模式虚拟测试类。

扩展

  工厂模式可以有多种扩展方式：

  • 简单工厂模式 / 静态工厂模式
  • 多工厂模式
  • 代替单例模式
  • 延迟初始化
